# AgentGuard — Design Context

## Users
Developers building AI agents that transact on-chain. They live in dark
IDEs, read tx hashes for sport, and judge dev tools by how fast and how
honest the dashboard feels. They want infrastructure-grade signal density,
not slick marketing surfaces.

## Brand Personality
**Precise · Opinionated · Utilitarian.**

The product is a security control plane for autonomous agents. It should
feel like a piece of operational infrastructure (Tailscale admin,
fly.io dashboard, Cursor's CLI panes) — unmistakably designed but never
decorative. Trust is built through transparency and information density,
not via gradients or hero illustrations.

## Aesthetic Direction
**Operations console, dark only.**

- Tunnel-warmed near-black background (`oklch(0.16 0.005 80)`), no pure
  black, no pure white. Subtle warm tint toward the accent so the palette
  hangs together.
- **Single accent: CRT amber** (`oklch(0.82 0.14 80)`). Used sparingly —
  active live indicator, primary CTA only. Status uses semantic green /
  red / pending-amber for tx state, not for branding.
- **Type**:
  - `Funnel Sans` (Google Fonts, free, geometric grotesque with personality)
    for prose and labels
  - `Martian Mono` (Google Fonts, NASA-derived, distinctive) for ALL data:
    addresses, hashes, amounts, keys, timestamps. Mono dominates because
    the product *is* the data.
- **No rounded cards.** 1px borders, near-square corners (max 4px radius).
- **No card-inside-card** nesting. Flatten hierarchy.
- **No gradient text. No left-stripe borders. No glassmorphism. No
  generic icon-above-heading layout.** (Standard absolute bans from the
  impeccable skill.)
- High information density: timestamps, tx hashes, status dots in a
  single row. Operations users want to scan, not scroll.

## Design Principles
1. **Data is the design.** Monospace data fields are first-class visual
   elements. Prose explanations are secondary.
2. **One accent, used rarely.** Amber means "active / primary action".
   When you see it, it matters.
3. **Show the chain.** Every address and hash links to basescan with no
   ceremony.
4. **Live state must look alive.** A breathing pulse dot, not a hover
   tooltip, is how we signal "this feed is polling right now".
5. **Honest about state.** A failed UserOp shows the raw on-chain error,
   not a sanitized "Something went wrong".
